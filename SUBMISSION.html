1. Entity Relationship Design

The system is built on two core entities: Author and Book. They are connected via a One-to-Many relationship.

  • Author: Master entity containing writer details.
  • Book: Transactional entity containing book details and a foreign key to the Author.

2. Implementation Details

A. Database Population

We used CommandLineRunner to seed the H2 database with 10 authors and 10 books on startup.

B. Create Operation

Handled by LibraryController.saveBook(). It accepts a Book object bound from the JSP form and saves it via the service layer.

@PostMapping("/save-book")
public String saveBook(@ModelAttribute("book") Book book, Model model) {
    try {
        libraryService.saveBook(book);
        return "redirect:/";
    } catch (Exception e) {
        model.addAttribute("error", "Error saving book: " + e.getMessage());
        return "form";
    }
}

3. Challenges Faced & Solutions

  • Dependency Management: Upgrading to Spring Boot 3 meant switching from javax.servlet to jakarta.servlet. This was resolved by updating the JSTL and Jasper dependencies in the POM.
  • Performance Optimization: To avoid the N+1 problem when listing books with authors, I implemented a custom JPQL join query in the repository layer.
